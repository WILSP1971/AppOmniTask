Guía de pruebas de la API

Los 26 endpoints que la app invoca de verdad, contra la API real de producción, con datos de ejemplo listos para pegar en la consola del navegador (F12) y validar por tu cuenta — sin instalar Postman ni nada aparte.

Cómo usar esta guía

  1. Abre cualquier página en el navegador y presiona F12 (o clic derecho → Inspeccionar) para abrir las herramientas de desarrollo, y ve a la pestaña Console.
  2. Pega el bloque de la sección 0. Variables primero — define la URL base y, más adelante, tu token.
  3. Empieza por Auth → Iniciar sesión (o Registrar, si es tu primera vez). La respuesta trae access_token — cópialo y pégalo en la variable TOKEN.
  4. De ahí en adelante, cualquier bloque que diga "Requiere sesión" ya manda ese token en el header Authorization automáticamente si seguiste el paso anterior.
  5. Cada bloque imprime la respuesta completa en la consola — revisa el código HTTP y el cuerpo JSON para confirmar que la API responde lo esperado.

0. Variables (pegar primero)

Define esto una vez por sesión de consola — se pierde si recargas la página.

const API = "https://appsintranet.esculapiosis.com/APIOmniTask/api/v1";
let TOKEN = ""; // lo llenas después de login/registro
let REFRESH = ""; // idem, con el refresh_token

const authHeaders = () => ({
  "Content-Type": "application/json",
  ...(TOKEN ? { Authorization: `Bearer ${TOKEN}` } : {}),
});

Auth

POST /auth/register Pública

Cambia el correo en cada prueba — repetirlo da 409 conflict (ya registrado).

Consola del navegador
fetch(`${API}/auth/register`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    full_name: "Usuario de Prueba",
    email: `prueba.${Date.now()}@clinicacampbell.test`,
    password: "ClaveSegura123",
    phone_e164: "+573001234567",
    timezone: "America/Bogota",
  }),
}).then(r => r.json()).then(d => { TOKEN = d.access_token; REFRESH = d.refresh_token; console.log(d); });
curl
curl -X POST "$API/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"full_name":"Usuario de Prueba","email":"prueba.123@clinicacampbell.test","password":"ClaveSegura123","phone_e164":"+573001234567","timezone":"America/Bogota"}'
POST /auth/login Pública

Usa el correo/contraseña de un usuario ya registrado.

Consola del navegador
fetch(`${API}/auth/login`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ email: "prueba.123@clinicacampbell.test", password: "ClaveSegura123" }),
}).then(r => r.json()).then(d => { TOKEN = d.access_token; REFRESH = d.refresh_token; console.log(d); });
POST /auth/refresh Pública*

*No lleva access token, pero sí exige un refresh_token válido en el cuerpo — de un solo uso, rota en cada llamada.

fetch(`${API}/auth/refresh`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ refresh_token: REFRESH }),
}).then(r => r.json()).then(d => { TOKEN = d.access_token; REFRESH = d.refresh_token; console.log(d); });

PATCH /auth/me Requiere sesión

Todos los campos son opcionales — solo manda los que quieras cambiar.

fetch(`${API}/auth/me`, {
  method: "PATCH",
  headers: authHeaders(),

POST /auth/logout Requiere sesión

Revoca el refresh_token — después de esto, /auth/refresh con el mismo token debe dar 401.

fetch(`${API}/auth/logout`, {
  method: "POST",
  headers: authHeaders(),
  body: JSON.stringify({ refresh_token: REFRESH }),
}).then(r => console.log(r.status));

Actividades

POST /activities Requiere sesión

type: meeting / appointment / task. Omitir starts_at crea la actividad "sin programar". meeting_url/meeting_provider son opcionales (provider: meet, teams u other). contact_ids (SPEC-008) acepta 0, 1 o varios contactos del usuario autenticado — un id que no exista o sea de otro usuario se ignora sin error; contact_id (un solo id) se conserva por compatibilidad con clientes viejos y, si llega junto con contact_ids, ambos se combinan. La respuesta trae tanto contacts/contact_ids como contact_id (el primero de la lista).

fetch(`${API}/activities`, {
  method: "POST",
  headers: authHeaders(),
  body: JSON.stringify({
    type: "meeting",
    title: "Reunión de prueba",
    description: "Creada desde la guía de pruebas",
    contact_ids: [],
    starts_at: new Date(Date.now() + 2 * 86400000).toISOString(),
    ends_at: new Date(Date.now() + 2 * 86400000 + 3600000).toISOString(),
    location: null,
    meeting_url: "https://meet.google.com/abc-defg-hij",
    meeting_provider: "meet",
  }),
}).then(r => r.json()).then(d => { console.log(d); window.__activityId = d.id; });
GET /activities Requiere sesión

Filtros opcionales: type, status (scheduled/completed/cancelled). from/to en UTC.

const from = new Date(Date.now() - 7 * 86400000).toISOString();
const to = new Date(Date.now() + 30 * 86400000).toISOString();
fetch(`${API}/activities?from=${from}&to=${to}&page=1&limit=50`, { headers: authHeaders() })
  .then(r => r.json()).then(console.log);
GET /activities/unscheduled Requiere sesión 
fetch(`${API}/activities/unscheduled`, { headers: authHeaders() }).then(r => r.json()).then(console.log);
GET /activities/{id} Requiere sesión

Usa el id que devolvió el POST /activities de arriba (queda guardado en window.__activityId).

fetch(`${API}/activities/${window.__activityId}`, { headers: authHeaders() }).then(r => r.json()).then(console.log);
PATCH /activities/{id} Requiere sesión

clear_starts_at/clear_ends_at distinguen "no tocar" de "quitar la fecha" — mándalos siempre (el ejemplo reprograma, por eso van en false).

fetch(`${API}/activities/${window.__activityId}`, {
  method: "PATCH",
  headers: authHeaders(),
  body: JSON.stringify({
    title: "Reunión reprogramada",
    starts_at: new Date(Date.now() + 5 * 86400000).toISOString(),
    clear_starts_at: false,
    clear_ends_at: false,
  }),
}).then(r => r.json()).then(console.log);
DELETE /activities/{id} Requiere sesión

Soft delete — equivale a status: cancelled, no borra la fila.

fetch(`${API}/activities/${window.__activityId}`, { method: "DELETE", headers: authHeaders() })
  .then(r => console.log(r.status));

Adjuntos

Subir un archivo necesita un selector real La subida es multipart/form-data — un fetch de una línea no puede adjuntar un archivo de tu disco por sí solo. El bloque de "Subir" de abajo crea un selector de archivo temporal en la página; el resto sí son fetch normales.
GET /activities/{id}/attachments Requiere sesión 
fetch(`${API}/activities/${window.__activityId}/attachments`, { headers: authHeaders() })
  .then(r => r.json()).then(console.log);
POST /activities/{id}/attachments Requiere sesión

Máximo 10 MB. Tipos permitidos: JPEG, PNG, HEIC, PDF. Pega esto y luego elige un archivo en el selector que aparece.

const input = document.createElement("input");
input.type = "file";
input.accept = ".jpg,.jpeg,.png,.heic,.pdf";
input.onchange = () => {
  const form = new FormData();
  form.append("file", input.files[0]);
  fetch(`${API}/activities/${window.__activityId}/attachments`, {
    method: "POST",
    headers: { Authorization: `Bearer ${TOKEN}` }, // sin Content-Type: el navegador pone el boundary
    body: form,
  }).then(r => r.json()).then(d => { console.log(d); window.__attachmentId = d.id; });
};
input.click();
GET /activities/{id}/attachments/{attachmentId} Requiere sesión

Devuelve los bytes íntegros del archivo — este bloque abre el resultado en una pestaña nueva para confirmar que se ve/descarga bien.

fetch(`${API}/activities/${window.__activityId}/attachments/${window.__attachmentId}`, { headers: authHeaders() })
  .then(r => r.blob())
  .then(b => window.open(URL.createObjectURL(b)));
